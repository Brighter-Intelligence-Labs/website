01 — Overview
Design Principles
Four rules that govern every decision in this brief. When in doubt, refer back to these.
Rule 1 — Left-align everything. Section headings, body copy, eyebrows, labels. Nothing is centred except the CTA block (which is intentionally an exception). Centred headings read as template output. Left-aligned headings read as editorial intent.
Rule 2 — No photography, no Unsplash. Every image slot is replaced with a data or structural treatment: stat blocks, pipeline diagrams, topology SVGs, metric grids. This is not a cost-cutting measure — it is a deliberate signal that the business thinks in systems, not vibes.
Rule 3 — Fluid spacing everywhere. All section vertical padding uses clamp(). No fixed pixel values for section spacing. This resolves the mobile/desktop rhythm problem permanently.
Rule 4 — Weight contrast over size contrast. Hierarchy is achieved by pairing weight 300 (body, subtitles) with weight 500–600 (labels, CTAs). Font sizes do not need to be dramatically large to read as primary.

03 — Typography
Type Scale
DM Sans throughout. No serif display font. Hierarchy is created through size, weight, and letter-spacing — not font switching.
Fonts: DM Serif Display (Google Fonts) for H1 and H2 only — weight 400. DM Sans for everything else — weights 300, 400, 500, 600. Load via: family=DM+Sans:wght@300;400;500;600&family=DM+Serif+Display. No other font families.

07 — Responsive
Breakpoint Rules
Breakpoint Width Layout changes
mobile < 640px Single column everything. Pipeline becomes vertical. Nav collapses to hamburger. Hero stack: copy then stat block. CTA button full-width.
tablet 640–1024px Service cards: 2-column. Insight cards: 2-column. Hero: single column. Differentiators: 2-column (unchanged).

08 — Constraints
What Not To Do
Explicit anti-patterns from the current site. Do not reproduce any of these.
No centred headings. text-align: center on any H1, H2, or H3 is prohibited. The only centred elements are within the CTA block.
No missing eyebrows. Every major section has an eyebrow label above its H2. Without it, sections feel anonymous.
No fixed section padding. padding: 80px 40px on a section is forbidden. Use clamp() on every section.
No Unsplash images. Not even "good" ones. The brand is systems-first. Imagery contradicts that.
No generic numbered lists for process. The "1. Discovery 2. Architecture..." text list is replaced with the pipeline diagram component.
No weight 400 on hero H1. The headline must be weight 300. The lightness creates the elegance.
